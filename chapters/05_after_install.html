絶対に覚える：シェル設定の用語

第4章で .zprofile にパスを通しました。意味が分からないまま進めた人もいると思います。ここで整理しておきます。

zshズィーシェル
一言で
Mac の最新版で標準採用されているシェル（コマンド翻訳係）の一種
たとえると
Mac に最初から入っている「ターミナル用の通訳さん」の名前。古い Mac は bash という別の通訳が標準だった
関連
bash / シェル / .zshrc / .zprofile
.zprofile / .zshrcドット ズィー...
一言で
zsh が起動時に必ず読み込む設定ファイル（ホームフォルダにある隠しファイル）
たとえると
「通訳さんの仕事マニュアル」。ここに「brew はこの場所にあるよ」など教え込んでおくと、毎回伝えなくて済む
関連
PATH / 隠しファイル（command + shift + . で表示切替）
PATHパス（環境変数）
一言で
シェルが「コマンドをどこから探すか」のリスト
たとえると
「brew って打たれたら、このフォルダ群を順に探して同名のプログラムを実行する」というルックアップ表。第4章でやった「パスを通す」とはここに登録すること
関連
環境変数 / export / 「command not found」
Hidden File隠しファイル
一言で
名前が .（ドット）から始まる、普段は表示されないファイル
たとえると
家の中の「設定の小箱」。普段は見えない方が散らからないので隠してある。.zprofile や .gitignore もこれ
関連
command + shift + . / ls -la
sudoスードゥ
一言で
そのコマンドを管理者権限で実行するための頭詞
たとえると
「マスターキーで強制的に開ける」モード。便利だが、間違えるとシステムを壊しかねない。意味を分かるまで使わないのが鉄則（第7章再確認）
関連

動いたあとに、まずやっておくこと

① Cursor の日本語入力を確認

Cursor の中で AI に質問する時、日本語入力が変換しっぱなしで Enter を押すと、変換確定だけで終わってしまうことがあります。 一度 Cursor の AI チャット欄に「こんにちは」など打ってみて、ちゃんと送信されるか確認しましょう。

② Claude Code を一度起動して画面に慣れる

第6章で詳しくやりますが、軽く触っておくのがおすすめ。 ターミナルで claude と打ってみて、画面が立ち上がること、/exit で終了できることだけ確認してください。

③ シェルの設定ファイルを把握しておく

Mac の最新版では、ターミナルは zsh（ズィーシェル）というシェルが動いています。 その設定ファイルが ~/.zprofile や ~/.zshrc です。 Homebrew を入れた時に ~/.zprofile にパスを書き加えました。

無闇に編集しなくて構いませんが、「自分の Mac の設定はここに書いてある」ということだけ覚えておきましょう。

陥りがちな落とし穴

落とし穴 1：開発フォルダの置き場所を間違える

最初は「分かりやすいから」とデスクトップに開発用のフォルダを作りたくなりますが、おすすめしません。

  • デスクトップは iCloud と同期されている場合があり、Mac が重くなる原因になる
  • ファイルが増えると視認性が落ち、誤って消しやすい
  • 書類フォルダ（~/Documents/）も同様に iCloud 同期対象になりがち

本書では、ホーム直下の ~/dev/ を採用します（~ はホームフォルダの省略記号）。 なぜここを推奨するのか・他にどんな選択肢があるのか（macOS の ~/Developer/ など）は、第7章「開発フォルダの適切な場所」で詳しく解説します。

また、フォルダ名は英数字＋ハイフンを強く推奨します（日本語名は GitHub や CLI で扱う時に思わぬトラブルの元になります）。 こちらも詳細は第7章。

落とし穴 2：ターミナルでコピペしたつもりが、形が崩れている

Web ページからコマンドをコピーすると、改行や空白が変な形になることがあります。 うまくいかない時は、コマンドをいったん「メモ」アプリに貼って、形を確認してからターミナルに貼り直してみてください。

落とし穴 3：エラーメッセージを読まずに連打する

コマンドが失敗した時、画面に出てくるエラーメッセージは 必ず読みましょう。 だいたい原因がそこに書いてあります。読んで分からない時は、そのまま AI にコピペして聞くのが最速です。

ターミナルで以下のコマンドを実行したら、こういうエラーが出ました。
初心者向けに、原因と対処法を教えてください。

実行したコマンド：
[コマンドを貼り付け]

エラー全文：
[エラーをそのまま貼り付け]

落とし穴 4：「とりあえず sudo を付ける」

sudo（スドゥ）は「管理者権限で実行する」という意味の頭詞です。 強い権限なので、間違えるとシステムを壊しかねません。

うまくいかないからといって sudo を付けて再実行するのは、初心者が一番やってはいけない癖です。
エラーが出たら、まず原因を AI に聞いて、それでも sudo が必要だと判断できた場合に限って使ってください。

落とし穴 5：何でもコマンドを試してしまう

Web 上には善意も悪意も含めて様々なコマンドが転がっています。 意味の分からないコマンドは絶対に実行しないのが鉄則です。 必ず最初に AI に「これは何をするコマンド？」と聞きましょう。

Mac 全体の設定で見直すと楽になるもの

設定箇所変更内容理由
Finder > 表示「すべての拡張子を表示」を ONHTML / CSS など拡張子の違いが見えると安心
Finder > 表示「パスバーを表示」を ON今いるフォルダの場所が常に見える
Finder > 環境設定「すべてのフォルダで表示」を ON（隠しファイル）.gitignore など . から始まるファイルが見えるようになる（command + shift + . でも切替可）
