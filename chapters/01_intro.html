CHAPTER 01

はじめに：この指南書の使い方

この章のゴール
この指南書が何のためのもので、どんな順番で読めばよいかが分かる。
自分が今どの位置にいて、最終的に何ができるようになるのか、見通しを持てる状態になる。

この指南書は誰のためか

この指南書は、株式会社 Poolside の仲間として、信頼を寄せている若手の育成を目的に書かれています。 Claude Code（クロード コード）とCursor（カーソル）を使いこなし、AI を相棒に開発を進められるようになることがゴールです。

社外公開を前提とした一般向けのチュートリアルではなく、あなた個人に向けた、上長からの実地の手引きとして渡しているものです。 だからこそ、遠慮のない物言いも、業務に直結する具体例も、率直に書いています。

前提知識は要りません。次のような状態の人を想定して書いています。

  • プログラミングをしたことがない
  • 「ターミナル」と聞いてもピンと来ない
  • Mac は普段使っているが、Finder（フォルダを開いたり閉じたりする画面）以外はほぼ触っていない
  • 「AI を業務に使う」と言われたが、何から始めればいいのか分からない

この指南書を読み終えると何ができるか

  1. Mac のターミナルを怖がらずに開けるようになる
  2. Claude Code と Cursor を自分の Mac にインストールできる
  3. Claude Code に話しかけて、簡単な作業を頼めるようになる
  4. HTML / CSS / JavaScript を AI と相談しながら書ける
  5. 作った Web ページを GitHub または Firebase で インターネットに公開できる
  6. AI を使う上で気をつけるべきこと（機密情報・リスク）が分かる

最後の項目は地味に見えますが、業務で AI を使うなら最重要です。第7章で詳しく扱います。

読む順番について

基本は 1章から順番に読むことをおすすめします。とくに第3章〜第7章までは、 後の実装練習で必要になる土台なので飛ばさないでください。

すでにターミナルや Mac の操作に慣れている方は、第3章を飛ばして第4章のインストールから始めても構いません。

実装練習（第8〜10章）は、頭で理解するより実際に手を動かすことが大事です。 この指南書を読みながら、Cursor や Claude Code の画面を別ウィンドウで開いて、コマンドや手順を真似してみてください。

本書のお約束

  • 命令口調にはしません。一緒に進めるつもりで書いています
  • 専門用語が出てきた時は、その場で1〜2行で噛み砕きます
  • 「フォルダ」と「ディレクトリ」は 「フォルダ」に統一します（実は同じもの）
  • コマンド例は そのままコピーして貼り付ければ動く形で書きます
  • うまくいかなかった時のフォローを各章末・第15章に用意しています
料金・有料プラン関連の判断は、必ず居村に相談してください。
本書を進める中で「Pro プラン」「Blaze プラン」「API 従量課金」などの選択肢が出てきます。 無料枠だけで進められるものを基本としていますが、業務で本格運用する段階では有料プランが必要になることもあります。 個人で勝手に有料プランへ切り替えないこと。事前に居村と相談して、契約・予算・利用範囲を決めてから動いてください。
（会社経費の取り扱い、退職時の引き継ぎ、セキュリティ面で関わる重要事項です）

大事な心得：コピペは入口、手打ちは目標

本書ではあえて、コマンドや AI への指示文（プロンプト）を コピペでそのまま動かせる 形で書いています。 最初の段階でタイプミスや細かい記号で詰まる経験は、学習意欲を削ってしまうからです。慣れるまでは堂々とコピペで進めてください。

ただし、本来の理想は簡単なコマンドや指示文を、自分の頭から自然に手で打てるようになることです。
コピペだけで進めていると、「何をやっているか」が頭に残りづらく、応用やトラブル対処の力が育ちにくくなります。

なので、本書を進めながら以下を意識してみてください：

  • 何度か出てきたコマンド（cd / ls / git add / git commit など）は、見ないで打てるよう少しずつ素振りする
  • AI に頼む時のプロンプトも、ある程度パターンが見えてきたら自分の言葉で書いてみる
  • 打ち間違えても何も壊れません。「打ち間違いを直す経験」が一番の上達近道です

最初は全部コピペでいい。慣れてきたら少しずつ手を動かす。半年後に「あの頃は全部コピペだったな」と笑えるようになるのが目標です。

大事な心得：知らない言葉が出たら、即 AI に聞け

本書はできるだけ平易に書いていますが、それでも知らない言葉や、よく分からない箇所は必ず出てきます。 なぜなら、あなたはまだ素人だからです。これは恥ずかしいことではありません。むしろ当然です。

ただし、いまの時代の素人と昔の素人では決定的に違うことがあります。 あなたには ChatGPT・Gemini・Claude といった超優秀な相棒がいることです。 分からない単語や概念に出くわしたら、考え込まず、1秒でも早く AI に聞いてください。 Google 検索で記事を漁るより、AI に聞くほうが圧倒的に早く・自分のレベルに合わせて答えてくれます。

「Homebrew」って何？小学生にもわかるように、たとえ話を交えて教えてください。
Mac の「パスを通す」ってどういう意味？なぜ必要なのかも含めて、絵で例えるように解説してください。
git の「コミット」を、料理の手順に例えて教えてください。

質問は恥ずかしがらないでください。むしろ、知らないことを即 AI に投げて潰せる人ほど、早く一人前になります。 ここで遠慮するより、聞きまくって 1 ヶ月で「分かる人」に化ける方が、結果的に評価されます。

AI に聞いて出てきた答えがピンと来なかったら、「もう一回、もっと簡単に」「中学生でも分かるように」「絵で描けるように」と同じ質問を表現を変えて投げ直しましょう。AI は何度聞いても怒りません。

つまずいた時

詰まったら、まず深呼吸してください。プログラミングは「動かない時間」のほうが「動いている時間」より圧倒的に長いものです。 そして、それを 解決していく過程そのものが学習になります。

本書の進行で詰まった時は、以下の順で対処してください：

  1. その章のページをもう一度ゆっくり読み返す（飛ばし読みしている可能性が大きい）
  2. 第13章「Q&A」に同じ症状がないか探す
  3. Claude Code 自体に「こういうエラーが出た」とそのまま貼り付けて聞く
  4. それでも解決しないときは、社内のメンター・先輩エンジニアに声をかける
AI に質問する時は、見栄を張らず、「初心者なので」「分からないので」と素直に書きましょう。AI はその前提を踏まえて返してくれます。
監修：居村より

この指南書を作って渡している、上長の居村です。Poolside の代表をやっています。

なぜわざわざ手作りの指南書を用意しているか、ひとこと書いておきます。

これからの仕事は、AI と組めるか・組めないかで生産性が 3 倍にも 10 倍にも変わる時代に入っていきます。 映像・音声・イベント・システムを横断する Poolside のような現場ではなおさらです。 AI を相棒にできる人ほど「考える時間」と「手を動かす時間」のバランスが取れて、本当にやりたい創造的な仕事に集中できるようになります。

ただし、AI 運用は最初の壁が高いのも事実です。ターミナル、npm、git——この入口で止まってしまう人が多すぎる。 せっかく良いセンスやアイデアを持っている人が、最初の段差で諦めてしまうのは、本当にもったいない。

だから、その最初の段差を一緒に超えるためのガイドとして、これを用意しました。 読み終わる頃には、あなたの仕事の進め方は確実に一段変わります。 そして、ここを超えた人だけが見える景色は、間違いなく面白いです。

ついでに、ぶっちゃけ自己紹介もしておきます。 私自身はモラハラ・パワハラ気質を多少引きずった昭和の男です。 ストレートすぎる物言いや、容赦のないツッコミに、最初は面食らうこともあると思います。 ただ、それは全部あなたが進化していくための「犠牲」だと思って受け止めてほしい。 甘やかして放っておくほうが、長い目で見ると失礼だと本気で信じているからです。 鍛えるべきところは厳しく、伸ばすべきところは惜しまず本気で。スタンスはそれだけです。

詰まったら遠慮なく聞いてください。一緒に詰まったところを潰していくのも、上司の仕事です。 最初は誰だって初心者でした。慌てず、自分のペースで、でも止まらずに進んでいきましょう。

— 居村
