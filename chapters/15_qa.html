CHAPTER 15

つまずいた時の Q&A

この章のゴール
よくあるトラブルを先回りして知っておく。
AI に質問するときのコツを身につけて、自力で抜け出せる場面を増やす。

その前に：AI に質問するコツ

① 「事実」と「感想」を分けて伝える

「うまくいきません」だけでは AI も状況が分かりません。次の3つに分けて伝えましょう：

  • 何をやろうとしたか（実行したコマンド・期待していた結果）
  • 実際に起きたこと（画面に表示されたメッセージ・出てきたエラー）
  • 自分の予想（「ここが原因かも」と思っていること、無くてもOK）

② エラーメッセージは そのまま貼り付ける

要約しようとして「エラーが出た」とだけ伝えるのは厳禁。 エラーメッセージには原因のヒントが詰まっているので、全文をそのままコピペしてください。

③ 自分のレベルを伝える

私は今この指南書の第◯章を進めている初心者です。
ターミナル経験はほぼ無く、専門用語は知らないので、その前提で教えてください。

④ 解決したら、何だったか自分にまとめる

解決したら、その日のうちに「何が起きていて、どう直したか」を一行メモしておきましょう。 数日後、同じ問題に当たった自分を救ってくれます。

よくあるエラーと対処

command not found

「打ったコマンドが見つからない」状態。原因は主に2つ：

  • スペルミス：claud（最後の e がない）など。打ち直す
  • インストールがされていない／パスが通っていない：第4章のインストール手順を再確認。brew --version や node --version から確認

permission denied

「権限がなくて実行できない」状態。よくあるパターン：

  • システムフォルダ（/usr/... など）を触ろうとしている → そもそも触らない方がよい場所
  • ファイルが読み取り専用になっている → ファイルの権限を見るか、Claude に状況を貼り付けて相談
sudo を付けて再実行するのは絶対に最後の手段。第7章を読み返してから判断してください。

No such file or directory

「そんなファイル／フォルダない」状態。

  • pwd（今いる場所を表示）で、自分の現在地を確認
  • ls で、いま見える範囲を確認
  • パスのスペルミス、大文字小文字、フォルダ階層の見間違いを疑う

fatal: not a git repository

「ここは git で管理しているフォルダじゃない」状態。

  • 正しい案件フォルダにいるか pwd で確認
  • 新規フォルダなら git init をする

「すでに使われています」「already exists」

同じ名前のファイル・リポジトリ・プロジェクトが既にある状態。別の名前を選ぶか、上書きしてよいか慎重に判断。

シーン別「困った」の対処

Claude Code が起動しない

  • claude --version で動くか確認
  • 動かないなら第4章のインストール再確認
  • 最新の Anthropic アカウントでログイン状態か確認（claude 起動時にエラーメッセージを読む）

Claude が会話を覚えていない

  • 違うフォルダで claude --continue を実行している → 元のフォルダに cd して再実行
  • 「No conversation found to continue」と出る → そのフォルダで初めて起動した状態。claude（オプション無し）で新規開始

ブラウザでファイルを開いても変更が反映されない

  • ファイルを保存したか確認（command + S）
  • ブラウザを強制更新（command + shift + R）
  • 正しい URL（フォルダ構造）を見ているか確認

「これ、勝手に書き換わったかも？」

  • git で履歴を取っていれば、いつでも戻せる：git log → git diff で確認
  • git の履歴がなければ Time Machine から戻せる場合あり
  • 慌てて何度も保存しない。状況を確認してから動く

AI に頼んだのに、想定と違う出力が返ってきた

  • 「いまのは取り消して、こうしたい」と追加注文を入れる
  • 頼み方を1つずつ・具体的に書き直す
  • 「以下の制約を守ってください」と前置きして、要件をリスト形式で提示する

会社のセキュリティで Homebrew が入らない

  • 管理者制限がかかっている可能性
  • 社内の情シス担当に「開発業務で Homebrew が必要」と相談

「どうしても解決しない」時のエスカレーション

順番に試してみてください：

  1. その章をもう一度ゆっくり最初から読み直す（飛ばし読みを疑う）
  2. 本章（第13章）の該当項目を読み直す
  3. Claude に状況を全部貼り付けて相談（事実 + エラー + やったこと）
  4. 社内のメンター・先輩エンジニアに連絡
  5. 情シス・社内ヘルプデスクへ
エラーで詰まった時間は「無駄」ではなく「経験値」。 解決した方法はメモして、次に同じことが起きたら3分で抜け出せる自分にしていきましょう。

もうちょっとレベルアップしたい時に

  • Claude Code の公式ドキュメントを眺める：https://docs.claude.com/claude-code
  • HTML / CSS の基礎を体系的に学ぶ：MDN Web Docs（https://developer.mozilla.org/ja/）
  • git の使い方をもう少し深く：https://git-scm.com/book/ja/v2
  • 困った時はまず Claude に「これ、どこから学ぶのが効率的？」と聞いてみる

最後にもう一回だけ言わせてくれ。 知らないことが出てきたら即 ChatGPT か Gemini か Claude に聞け。 キーボードは見るな、トラックパッドは触るな、トラックボールで動け。 この 2 つを徹底するだけで、3 ヶ月後の景色が変わる。本当に変わる。

逆に、これを軽視して半年後に「なんで自分は伸びないんだろう」と悩むやつを、何人も見てきた。 だから今、しつこく言っている。

ここまで読んだあなたへ

お疲れ様でした。最後まで読み切ったあなたは、もう「ターミナル未経験者」ではありません。 AI と一緒に開発できる人の 入り口に立っている状態です。

ここから先は、実案件で実際に手を動かして、失敗を重ねて、その都度 AI に聞いて、自分のメモを増やしていく。 そうやってあなた自身の運用スタイルができていきます。

Poolside での開発業務、楽しんでください。
