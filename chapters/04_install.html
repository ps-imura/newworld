CHAPTER 04

インストール手順

この章のゴール
自分の Mac に Cursor と Claude Code をインストールできる。
ターミナルから claude コマンドを打って AI を呼び出せる状態になる。

インストールするもの一覧

この章で入れていくのは次の4つです。

順ツール役割
1Cursorコードを書くエディタ（アプリ）
2HomebrewMac でツールを入れるための「アプリの自販機」のようなもの
3Node.jsClaude Code が裏で必要としている JavaScript の実行環境
4Claude Code本書の主役。AI と対話するためのコマンド
インストール作業には30分〜1時間かかることがあります。途中でパスワードを求められることもあります。 時間に余裕がある時にやりましょう。

① Cursor をインストール

  1. 公式サイトを開く
    ブラウザで https://cursor.com にアクセス。
  2. ダウンロードボタンを押す
    ページ上の「Download」ボタンを押すと、Mac 用のインストーラ（.dmg ファイル）がダウンロードされます。
  3. .dmg を開いて、アプリを Applications フォルダにドラッグ
    ダウンロードした Cursor.dmg をダブルクリックすると、Cursor のアイコンと Applications フォルダのアイコンが並んで表示されます。 Cursor のアイコンを Applications フォルダにドラッグ&ドロップしてください。
  4. 初回起動と Mac の許可
    Launchpad またはアプリケーションフォルダから Cursor を起動。「インターネットからダウンロードされたアプリです」という確認が出たら「開く」を選択。
  5. サインイン
    初回起動時に Cursor のアカウント作成 or サインインを求められます。会社の支給アカウントがあればそれを使い、無ければまずはご自身のメールで登録してから社内担当に連絡してください。

② Homebrew をインストール

Homebrew（ホームブリュー）は、Mac でツールをインストールするための公式風ツールです。 本書ではこのあと出てくる git や node を入れる時にも使います。

  1. ターミナルを開く（第3章を参照）
  2. 下のコマンドをコピーして、ターミナルにペーストして Enter 
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

途中で Mac のログインパスワードを聞かれるので入力（打っても画面には表示されません）。 インストールには5〜10分かかります。

パスを通す

インストールが終わると、画面の最後に「Next steps」のような英文と、もう2つ実行すべきコマンドが書かれています。 通常、次のような内容です（自分の画面の指示通りにコピーしてください）。 

echo >> ~/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

コピペ実行したら、ターミナルを一度閉じて開き直してください。 そのあと次のコマンドで動作確認します： 

brew --version

Homebrew 4.x.x のように表示されたら成功です。

画面に出ているメッセージは 毎回必ず読みましょう。「次にこれをやってね」と書いてあることが多いです。

③ Node.js をインストール

Claude Code は Node.js（ノードジェイエス）という基盤の上で動きます。Homebrew で入れます。 

brew install node

動作確認：

node --version
npm --version

どちらもバージョン番号（例：v22.x.x）が表示されれば OK です。

④ Claude Code をインストール

いよいよ本書の主役を入れます。 

npm install -g @anthropic-ai/claude-code

-g は「グローバルに（どこからでも使える形で）入れる」という意味です。 インストールには2〜5分かかります。

動作確認

claude --version

バージョン番号が表示されれば成功です。

Claude Code のインストール方法は今後変わる可能性があります。うまく動かない時は、必ず公式ドキュメント（https://docs.claude.com/claude-code など）を確認してください。 本書は2026年5月時点の情報をもとに書いています。

⑤ 初回ログイン

Claude Code を初めて使うときは、Anthropic アカウントでのログインを求められます。 任意の場所でターミナルを開き、次を実行： 

claude

画面の指示に従って、ブラウザでログインします。 会社支給のアカウントがあれば必ずそれを使ってください（個人アカウントと混ぜない）。

git も入れておく（GitHub 編で使用）

第11章で GitHub を扱う時に必要なので、ついでに入れておきます。 

brew install git
git --version

バージョンが出れば OK。

つまずきやすいポイント

  • 「command not found: brew」：パスを通すコマンドを実行していない／ターミナルを開き直していない（②の最後を再確認）
  • パスワードが通らない：Mac のログインパスワードを入力。Apple ID のパスワードではない
  • npm install で permission denied：管理者権限が必要なコマンドかも。sudo を頭に付けたくなるが、その前に第7章を読んでから判断してください
  • 会社の Mac で Homebrew が入れられない：管理者制限の可能性あり。社内の情シス担当に相談してください

ここまで来たら：以後のターミナル操作は Cursor 内蔵がおすすめ

第2章で触れた通り、Poolside の運用は 「Cursor を起動して、その内蔵ターミナルで claude を動かす」のが標準です。 Cursor を入れた今、以下の流れに慣れていきましょう：

  1. Cursor を起動する
  2. 「File → Open Folder」で案件フォルダを開く
  3. control + `（バッククォート）で内蔵ターミナルを開く
  4. そこで claude や brew や git を実行する

こうすると、Cursor で開いているフォルダが自動的にターミナルのカレントディレクトリになるので、cd し忘れの事故が起きません。 本書では第6章以降、この運用を前提にしていきます。
